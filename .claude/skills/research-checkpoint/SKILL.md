# Research Checkpoint

**This skill is scaffolded and needs to be designed through a live session — similar to how research-framing was built.** The interaction pattern, phase structure, and prompt deployment logic should be developed by walking through a real mid-study scenario.

## Intended Purpose

Sanity-check emerging patterns, pressure-test hypotheses, identify evidence gaps, and course-correct during active fieldwork. This skill operates during Hall's Analysis and Synthesis stage (while data collection is still ongoing).

## Nikki Anderson Prompts to Integrate

Detailed prompt content should be added to references/nikki-checkpoint-prompts.md. The following prompts from Nikki's article map to this stage:

- **Mid-Study Insight QA:** For catching overinterpretation, missing evidence, and premature conclusions
- **Hypothesis Stress-Test:** For checking whether working hypotheses are durable
- **Friction Spotting:** For articulating observed friction with precision (cognitive, emotional, navigational, trust, value, expectation mismatch, effort)
- **Mid-Flight Course Correction:** For when something has changed (prototype, scope, external context) and the plan needs adjusting

## Key Calabro Application

At this stage, the critical question is: "What have we actually captured so far — Category A behavioral evidence, or mostly Category B and C?" This prevents the user from building confidence on weak evidence mid-study.

## Design Notes for Future Build

- The reflect-back pattern is especially important here because mid-study thinking tends to be fragmented
- The skill should help distinguish between "I'm seeing a real pattern" and "I'm anchoring on the first few sessions"
- Alternative explanations for observed patterns should be surfaced proactively
- The skill should ask what evidence would *weaken* the current hypothesis, not just what supports it
